The Problem

Data was rich. Access was brittle.

The enterprise had invested years building a 10PB+ multi-cloud data lake — real-time telemetry, network analytics, operational intelligence across 300+ global venues. The data was there. The problem was access.

Engineers had to write SQL. Business users couldn't query at all. And as AI agents were deployed across the organization, they had no standardized way to reach live production data — every integration was bespoke, fragile, and required engineering cycles to maintain.

The broader AI transformation was blocked by a fundamental architectural gap: agents couldn't talk to data without custom plumbing for every use case.

The Approach

One interface. Engineers and agents, unified.

The solution was to build a Model Context Protocol (MCP) layer on top of BigQuery — a standardized interface that exposes the enterprise data lake to both human engineers and AI agents through the same access pattern.

Rather than building separate integration paths for each agent or use case, the MCP layer acts as a universal data contract: structured, governed, and queryable by anything that speaks the protocol. This means an agentic ITSM platform, a network self-healing agent, and a human engineer all reach production data through the same interface — with the same governance controls applied consistently.

For Engineers
Natural language → SQL → Results

Engineers query the data lake without writing complex SQL from scratch. The MCP layer translates intent into governed queries against live production data.

For AI Agents
Agents query live data autonomously

Autonomous agents deployed across the enterprise — ITSM, network ops, customer care — pull real-time production data directly as part of their decision loops, without human-in-the-loop data retrieval.

Key Decisions

Why this, why now

Protocol
Standardize on MCP before it was mainstream. The decision to adopt Model Context Protocol early was about future-proofing — building an access layer that would work with any agent framework, any LLM, as the ecosystem matured. Avoided vendor lock-in at the data access layer.
Governance
Agents get the same governance humans get. A common mistake in agentic architectures is treating agent data access as a special case with looser controls. Every query — human or agent — passes through the same identity and access management layer. Compliance stays intact.
Foundation
BigQuery as the single source of truth. After migrating the full data lake from Snowflake to GCP BigQuery, building the MCP layer on BigQuery made the access interface stable and consistent — agents always reach the same governed, production-grade data foundation.
Design
Reusable pattern, not a one-off integration. Built the MCP layer as a reusable platform component — any new agent deployed across the enterprise inherits data access on day one, without custom engineering per use case.
